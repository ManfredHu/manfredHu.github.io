# HTML meta 元素整理与实战

meta 的本质是给浏览器、爬虫和系统容器提供页面级元信息。它不直接渲染 UI，但会影响 SEO、移动端显示、社交分享、PWA 体验和安全策略。

先记一个原则：

- 能用标准标签就不用私有标签
- 能删掉历史包袱就删掉
- 每个 meta 都要有明确收益

---

## 一套推荐 head 模板

```html
<head>
	<meta charset="UTF-8" />
	<meta name="viewport" content="width=device-width, initial-scale=1" />
	<meta name="description" content="页面描述，建议 70-160 字" />
	<meta name="theme-color" content="#0f172a" />

	<!-- iOS Safari -->
	<meta name="apple-mobile-web-app-capable" content="yes" />
	<meta name="apple-mobile-web-app-status-bar-style" content="default" />
	<meta name="format-detection" content="telephone=no, email=no" />

	<!-- Open Graph -->
	<meta property="og:type" content="website" />
	<meta property="og:title" content="页面标题" />
	<meta property="og:description" content="页面描述" />
	<meta property="og:image" content="https://example.com/cover.png" />

	<!-- Twitter Card -->
	<meta name="twitter:card" content="summary_large_image" />

	<!-- 可选：基础 CSP（推荐改为服务端响应头） -->
	<meta
		http-equiv="Content-Security-Policy"
		content="default-src 'self'; img-src 'self' https: data:; script-src 'self'; style-src 'self' 'unsafe-inline'"
	/>
</head>
```

## 常见 meta 元素速查

### 1) 编码与视口

```html
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1" />
```

- `charset` 建议放在 head 最前面，避免乱码
- `viewport` 推荐保持简洁，不建议默认禁用缩放（`user-scalable=no` 会影响无障碍）

### 2) SEO 与索引控制

```html
<meta name="description" content="页面摘要" />
<meta name="robots" content="index,follow" />
```

- `description` 常用于搜索摘要
- 需要阻止收录时可用：`noindex,nofollow`

### 3) 主题色与浏览器 UI

```html
<meta name="theme-color" content="#ffffff" />
```

- Android Chrome、部分移动浏览器会用它渲染地址栏颜色

### 4) iOS Web App 相关

```html
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
<meta name="format-detection" content="telephone=no, email=no" />
```

- 添加到主屏后的 iOS 页面常用
- `format-detection` 可关闭自动电话/邮箱识别

### 5) 社交分享（Open Graph / Twitter）

```html
<meta property="og:title" content="文章标题" />
<meta property="og:description" content="文章摘要" />
<meta property="og:image" content="https://example.com/cover.png" />
<meta property="og:url" content="https://example.com/post/1" />
<meta name="twitter:card" content="summary_large_image" />
```

- 决定微信/Telegram/Slack/X 等渠道的分享卡片效果

### 6) 安全相关（可选）

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'" />
<meta http-equiv="Referrer-Policy" content="strict-origin-when-cross-origin" />
```

- 更推荐在服务端响应头设置；meta 方式适合静态托管场景兜底

## 历史标签与兼容项（了解即可）

下面这些标签在历史项目里很常见，但大部分已经不建议新项目继续使用：

```html
<!-- 避免 IE 进入兼容模式（仅历史项目） -->
<meta http-equiv="X-UA-Compatible" content="IE=edge" />

<!-- 360 浏览器内核选择（私有） -->
<meta name="renderer" content="webkit" />

<!-- 旧移动设备优化（基本淘汰） -->
<meta name="HandheldFriendly" content="true" />
<meta name="MobileOptimized" content="320" />

<!-- UC / QQ 私有全屏、横竖屏（非常旧） -->
<meta name="screen-orientation" content="portrait" />
<meta name="x5-orientation" content="portrait" />
<meta name="full-screen" content="yes" />
<meta name="x5-fullscreen" content="true" />
<meta name="browsermode" content="application" />
<meta name="x5-page-mode" content="app" />
```

建议：除非你要维护存量业务且目标浏览器明确包含这些内核，否则直接删掉。

## 别混淆：这些不是 meta

下面这些能力常和 meta 一起出现，但属于别的标签/属性：

- DNS 预解析：`<link rel="dns-prefetch" href="//example.com" />`
- 资源预取：`<link rel="prefetch" href="/next-page" />`
- 预渲染：`<link rel="prerender" href="/next-page" />`
- 下载属性：`<a href="file.pdf" download>下载</a>`
- 表单 pattern：`<input pattern="..." />`
- datalist 自动补全：`<datalist>...</datalist>`

它们很有用，但严格来说不属于 meta 元素。

## 实战建议

### 1) H5 页面最小集合

```html
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1" />
<meta name="description" content="..." />
<meta name="theme-color" content="#fff" />
<meta name="format-detection" content="telephone=no, email=no" />
```

### 2) 内容站/博客集合

在上面的基础上补齐：

- `og:title`
- `og:description`
- `og:image`
- `og:url`
- `twitter:card`
- canonical（`<link rel="canonical" ...>`）

### 3) 管理后台集合

除了基础项，建议优先配置 CSP、Referrer-Policy，并限制第三方脚本来源。

---

## 小结

meta 不是写得越多越好，而是写得越准越好。现代页面的核心通常就几项：

- `charset`
- `viewport`
- `description`
- `theme-color`
- Open Graph / Twitter Card
- （可选）CSP / Referrer-Policy

其余历史私有标签尽量按需保留，不要无脑复制模板。
