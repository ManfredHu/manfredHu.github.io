# 外设进化之路

> 总结一下现在的配置: **无线HHKB + 无线Logic Master3 + 触摸板(偶尔) + Airpods(上下班) + Bose35 II(公司开会)**
> 
> **优势**: 无线套装, 大鼠标好掌握. HHKB熟悉了不会再想换其他键盘, 太省心效率太高了. 两个耳机切换着用耳朵不会痒, 续航也很足

> **劣势**: Mac偶尔会有太多蓝牙键盘链接导致开会电麦, 原因是Mac连接太多蓝牙设备, 信道冲突. 解决办法就是鼠标用无线接收器.

从高中打游戏就很迷恋键盘, 当时因为玩Dota需要多线操控对机械键盘就很迷恋. 
看到外设这个词可能大家会认为机械键盘，不是都是通用的吗？我改下键不就好了？但是我却不这么认为，一般的机械键盘，对于MacOS来说，其实是很难匹配的。你会发现Windows的键位和Mac还是有很大出入的。

不适应大概有这么几个：
1. 键位不一致, 其实Mac的键位对比古老的键盘是更传统的,也是更符合程序员习惯的.而Windows则不然
2. 需要花费时间和精力去设置自定义按键
3. 对键盘会形成自己的手指肌肉记忆，经过一段长期的训练的，人会慢慢熟悉键盘
4. 按键需要有段落感, 提升自己的打码体验，但是又不能花里胡哨, 比如一些游戏机械键盘很炫酷, 但是个人不喜欢华而不实的东西

### 2022-1 HHKB

闲下来有时间折腾一下外设，传说中的程序猿的终极键盘，我也了解一些历史，比如HHKB跟苹果早起键盘布局其实是很像的。如下图
![](https://raw.githubusercontent.com/ManfredHu/manfredHu.github.io/master/images/../efficency/Apple II)[../images/keychron/appleII.jpeg]

HHKB键盘，作为一款深受程序员与设计师喜爱的键盘，以其独特的键位布局、高质量的手感和精湛的工艺脱颖而出。HHKB键盘全名为Happy Hacking Keyboard，是一款专为程序员和设计师设计的键盘。与传统键盘相比，HHKB的键位布局有很大区别：

1. 60%键盘布局：HHKB采用了60%的键盘布局，即去掉了数字键盘、功能键行和导航键区，使得键盘体积更小、携带更方便。
2. Control键位置：在HHKB键盘上，Control键被移动到了Caps Lock键的位置，更便于程序员使用。
3. ESC键与Backspace键位置：ESC键被移动到了`键的位置，而Backspace键移到了原ESC键的位置，使得这两个常用键更容易触及。
4. 分离的方向键：HHKB的方向键被集成到了Fn键与其他键的组合中，可以通过Fn+其他键实现上下左右的移动。

关于HHKB的试用, 你可以看这里 [HHKB快乐键盘](./hhkb.md)

#### HHKB键盘型号及价位对比
HHKB键盘有多个型号，这里我们主要对比三个较为常见的型号：HHKB Pro Classic、HHKB Pro Hybrid和HHKB Pro Hybrid Type-S。

- HHKB Pro Classic：作为基础款，这款键盘拥有基本的HHKB键位布局和PBT键帽。价格相对较低，适合初次接触HHKB的用户。
- HHKB Pro Hybrid：在Classic基础上，增加了蓝牙功能，支持多设备无线连接。价格略高于Classic，适合需要在多个设备间切换的用户。
- HHKB Pro Hybrid Type-S：除了拥有Hybrid的特性外，Type-S还具备静音功能，键盘敲击声更低。价格最高，适合对噪音敏感的用户。

推荐后面两款, 无线的爽谁用谁知道. 连接挺稳定的.

#### 使用体验
在经过两天的熟悉之后，我发现HHKB键盘布局确实需要较高的学习成本来适应。而一旦习惯了HHKB，重新适应其他键盘可能会变得困难，这就容易让人陷入“围城”的境地。然而，熟悉键位后，我发现更少的键位实际上能让我更专注于手头的工作，减少手腕的移动。特别的需要一些时间学习Vim键位, HHKB+Vim真的是效率伴侣

科学上说，我们现在使用的键盘布局起源于欧美，基本上是美式键盘布局, 也就是说其实代码, 英文文章更适合现有的键盘。而不是敲击拼音, 拼音是键盘后的产物, 我们熟悉了美式键盘的键位, 也熟悉了拼音输入法. 理解上总有一种别扭, 像是缝合怪. 体会不到原味的“爽”, 而HHKB+Vim的搭配可以体验老外的体验了.

逐渐过渡到Vim的键位后，我发现使用VSCode的Vim插件确实能提高编码效率，快捷键用得很顺手。但是，当我同时运行多个程序时，尤其是在进行小程序开发时，16G内存经常被占满，导致Vim插件无响应。遇到这种情况多次后，我放弃了Vim插件，转而使用Mac原生支持的EMacs快捷键进行光标移动。目前来看，效果不错。

至于现有外设，我使用了无线黑色HHKB（新款支持蓝牙4.2连接，更稳定且省电，不建议购买3.0版本，据说存在问题）、雷蛇有线鼠标以及一个剩余的MagicTrackpad。这些设备暂时足够使用。触摸板是公司福利，我发现习惯后用于窗口和程序切换非常顺手。当然，如果将键盘放在笔记本上，笔记本自带的触摸板也足够使用。我之所以采用MagicTrackpad，是因为在家中显示器被柜子顶部挡住，无法升高，放在笔记本下会遮挡视线，所以开始居家办公后购买了一个触摸板。

如今，我的追求已经从外设越多越好转变为越少越好，这有点像退烧的感觉。这一点与HHKB的设计理念相符：**越简单稳定越好**。电脑软件每年更新，而外设与你的匹配程度则是恒定的。基于这套设备，我打算在未来三年内不再更换，因为它们基本够用了。

![](https://raw.githubusercontent.com/ManfredHu/manfredHu.github.io/master/images/keychron/IMG_0243.jpg)


还有HHKB被诟病最多的方向键，因为没有方向键的存在所以需要用`Ctrl+f/b`做光标的前后移动，`Ctrl+n/p`做光标的上下切换，这个只能在输入的时候才有用，如果是VSCode或者其他的上下左右选择其实是无效的。

综上：HHKB适合一开始就用上的同学，而且后面大概率跟其他键盘切换使用其实是冲突的。HHKB最好我认为是它的轴体，确实比一般的机械键盘柔和很多。

如果你习惯把键盘架在笔记本上，可以用Karabiner，有个选项当你笔记本连上了配置的键盘，笔记本内置的键盘会失效，这样键盘架笔记本上不会误触。体验拉满

## 触摸屏
其实对于触摸屏和鼠标, 个人认为是有争议的. 争议的点在于支撑性和效率, 鼠标效率高, 但是一些时候触摸屏效率也不差. 比如Mac的一些三指操作, 效率拉满, 鼠标很难匹敌.

但是长时间使用触摸板发现手会酸, 所以我抛弃了 MagicTrackpad 换回Logic Master3, 因为键盘架在电脑上的原因, 离自带的触摸板也很近. 精简一下配置放弃 MagicTrackpad 而用电脑自带的. 体验还行, 可能问题大的就是有外接显示器+合上笔记本的需求时候无法满足
