## 文本对象

比如`diw` ，iw可以理解为文本对象，而d是剪切删除，所以就是剪切文本对象，文本对象可以有多种组成包括引号内, 直到某个字符。

- **iw(**inside word**)** 当前单词
- **aw(around word)** 当前单词以及空格，符号等.
- **i(,i[,i{** 括号内的内容, 函数内 **ci{** 直接修改整个函数体
- **a(,a[,a{** 括号以及括号内的内容**`.a[0].`**

其中，`i(` 和 `a(` 中的括号可以换成任何成对的符号，例如 `i[`、`i{`、`i"` 等，表示成对的符号范围的内容。

如`const { loading, setLoading } = useSome()` 可以用 **di{** 删除括号内内容剩余`const {} = useSome()`

如`const { loading, setLoading } = useSome()` 可以用 **da{** 删除括号内容包括括号剩余`const  = useSome()`

### 英文文本组成

- **word 单词**
    - word 单词, 很纯粹的单词, 标点符号也算一个word
    - WORDs, 以空格和换行符为分割
    - word和WORDs区别
        ![](https://raw.githubusercontent.com/ManfredHu/manfredHu.github.io/master/images/vim/vim-word-vs-WORD-boundaries.png)
- **sentence 句子** 以句号（.）、感叹号（!）或问号（?）结束等称为sentence, 而不是换行空格tab
- **paragraph 段落** 以空行起止的一块区域, 也就是 `空行 内容 空行`这种结构的内容部分
- **section 节**，多个paragraph拼起来的，以换行符号开头的.
    - VSCode装了VIM插件后可以通过`Shift + []` 在section间跳转

[Vim documentation: intro](http://vimdoc.sourceforge.net/htmldoc/intro.html#%3C%3E)


## 组成拆分

### 动作

- **`d`** delete 删除
- **`r`** replace 替换
- **`y`** yank 复制
- **`v`** visual 选择
- **`s` 删除当前字符串并进入编辑模式, 等于x+i / `S`删除整行并插入, 等于dd+O**

### 介词（修饰符）

- **`f`** find 直到某个字符（包括该字符）/  `F` 反向
    - **`f`** 命令可以用来在当前行中查找某个字符，并把光标定位到这个字符的位置。例如，如果你想查找当前行中的字母 "a"，你可以按下 **`fa`** 命令，然后 Vim 就会把光标移动到第一个 "a" 字母的位置。如果你想找到下一个 "a"，可以继续按下 **`;`** 键，这样 Vim 就会继续在当前行中查找下一个 "a"。
- **`t`** till 直到某个字符（不包括该字符）/ `T` 反向
    - **`t`** 命令可以用来定位到某个字符的**前一个位置**。例如，如果你想把光标定位到第一个 "a" 字母的前一个位置，你可以按下 **`ta`** 命令。这样 Vim 就会把光标定位到 "a" 前面的一个位置。如果你想找到下一个 "a" 的前一个位置，可以按下 **`,`** 键，这样 Vim 就会继续在当前行中查找下一个 "a" 的前一个位置。
- **`i`** inside 表示在...之内
- **`a`** around 表示环绕

### 名词

- **`w`** word 一个单词
- **`s`** sentence 一个句子
- **`p`** paragraph 一个段落

### 组合 （**动词 + 介词 + 名词）**

- 删除当前的单词 : **`diw`** (delete inside word)
- 改变当前的句子：**`cis`**(change inside sentence)
- 删除文本直到字符e : **`dte`** (delete till 'e')
- 选择一个句子：**`vis`** (visual inside sentence)
- 重写函数体: **`ci{`** change inside char {

# 移动

## **屏幕移动**

- `control + f` next screen / `control + d` next half screen
- `control + b` prev screen / `control + u` prev half screen
- `gg` 第一行
- `G` 最后一行 / `<number> G` 移动到第n行
- `<number> + 回车` 往下移动10行
- 【没卵用】`<number> + 空格` 往前移动10个字符 vscode不行
- `HML` 移动到本屏幕上中下位置

## 字符移动基础

- `hjkl` 对应 ←、↓ 、↑、→，10j表示向下10行
- `x` 向后删除字符 / `X`向前删除字符
- `dd` 剪切整行
- `gg` 到首行 / `G` 到尾行
- `u` undo撤销 / `U` redo撤销的撤销
- `r` rewrite
- `o` 下行插入 / `O` 上行插入
- `f` 向前 / `F` 向后，比如fc直接跳到下一个c出现的地方
- `I/0/^`  行首 , `A/$` 行尾
- **`*` next indent 下一个同单词出现的地方 / `#` prev indent 上一个同单词出现的地方**

## 词间移动

- `w` 移动到下一个word词首, 符号也算one word.
- `W` 移动到下一个WORDs词首, 不包括标点符号
- `b` 移动到上一个单词词首（跟w相反）, 符号也算one word
- `B` 移动到下一个WORDs词首, 不包括标点符号
- `e` 移动到下一个word词尾, 符号也算one word
- `E` 移动到下一个WORDs词尾, 不包括标点符号
- `ge` 移动到上一个word词尾（跟e相反）, 符号也算one word
- `gE` 移动到上一个WORDs词尾（跟e相反）, 不包括标点符号

`W`，`B`，`E`以空格换行为分割的移动

## 高级移动

**`%` 在配对的 `()` `[]` 之间移动**

`H` `M` `L` 移动到当前窗口（不滚动）头部，中间，尾部, 配合屏幕滚动和`/` 查找快速定位关键词

**`G` 到文件的尾部，前面添加数字再按 `G` 跳到输入的行**

`{` `}` 跳到 前一个 / 后一个 空行

`(` `)` 跳到当前句子的 ****首 / 尾 

【没卵用】`-` `+` 跳到上一行，下一行

`{} = shift + []` 两者含义一直, 都可以在代码块间移动, 比如代码以空行分割

## `{}` 大括号跳转

> 💡 **建议使用%跳转, 其他改动太复杂** 
> 
> 对JS代码来说
> - `[]` 跳转到上一个 `}` 位置
> - `][` 跳转到下一个 `}` 位置
> - `[[` 文件首行
> - `]]` 文件尾行

**`[[` `]]` `][` `[]` 第一个字符是方向，第二个字符是匹配的字符.** 

【没卵用】默认 `[[` `]]` 都是跳转到首列 { 符号的。说白了就是给C语言用的，因为C语言规范{都在首列

【没卵用】默认 `][` `[]` 都是跳转到 } 符号的。JS里 } 一般单独一行, 偶尔可以用

 如果语法跟C语言不一样，{}不是在首列的，做如下代码修改

```bash
:map [[ ?{<CR>w99[{
:map ][ /}<CR>b99]}
:map ]] j0[[%/{<CR>
:map [] k$][%?}<CR>
```

# 单字符替换 r (replace) + <some>

rebase操作的时候经常需要替换前面的单词 `pick`→ `f` ，也就是合并某些commit

可以使用ciw (change inside word)，此时会删除word保留hash前一个空格进入编辑模式，输入`f` 也就是fixup就OK了

```bash
git rebase -i HEAD~10 # rebase前10次commit
```

`r`只会替换一个字符串

`R`会一直前进替换，每替换一个光标就往前走

**`~`** 可以对字符进行大小写切换，比如首字母

# 寄存器register

**寄存器（register）：用于存储 Vim 命令执行的结果，以便于之后复制、粘贴等操作。例如，你可以使用 `"ayiw` 命令将当前光标所在的单词复制到名为 `a` 的寄存器中，然后使用 `"ap` 命令将该单词粘贴到光标所在位置。这里`a` 就是一个命名寄存器**

复制粘贴删除都与register有关，默认vim的内容放在register里而不是系统剪贴板。register是系统默认剪贴板的超集

可以输入 `:reg` 查看寄存器内容

![](https://raw.githubusercontent.com/ManfredHu/manfredHu.github.io/master/images/vim/vim-registers-reference-table.png)

- `"yiw` 复制inside word到默认寄存器
- `"+p` 粘贴上一次寄存器剪贴板内容
- `*+p` 粘贴系统剪贴板

# 标记

**标记（mark）：用于标记文本中的某个位置，以便于之后在文本中快速跳转。**

例如，你可以使用 `ma` 命令在当前位置设置一个名为 `a` 的标记，然后使用 `'a` (~波浪号同个键位)命令快速跳转到该标记所在的位置。

# 复制粘贴

如果要从系统复制文本到vim就用`+p` 

## 复制 y (yank)

- `yy` 复制一行（连带整行空格等）
    - `5yy` 复制后面5行
- `y$` 光标处复制到行尾
- `^y$` 整行复制（忽略头尾空格等）
- `ggyG` 拷贝整个文件
- `+y` 复制内容到系统剪切板
- `yw` 复制一个单词 (包括单词后面的空白字符)
- `ye` 复制一个单词

## paste

- `p` 在光标位置之后粘贴
- `P`在光标位置之前粘贴
- `+p` 将剪切板内容粘贴到光标后面
- `+yy` 复制当前行到剪贴板

# 选择 v（visual）

visual模式非常适合选择，v进入visual模式

**V是选择一行，**快速选择多行可以这么来

1. **V**选择一行
2. **j**向下扩展到最后一行
3. **d**将内容剪切并存入寄存器
4. **p** print寄存器内容

**选择多列（块选择模式）可以用`ctrl + v`** ，比如多行编辑的时候

# 查找

`*` 往下查找光标单词

`#` 往上查找光标单词

`/` 符号进入顺序查找模式，配合 `n` 查找下一个，`N` 查找上一个

`?` 符号进入逆序查找模式，`n` 此时变为查找上一个，`N` 为查找下一个

`q/` 查看查找历史

# 替换

`:s`（substitute）命令用来查找和替换字符串。语法：`:{作用范围}s/{目标}/{替换}/{替换标志}`

- 当前行 `:s/foo/bar/g`
- 全文 `:%s/foo/bar/g`
- 选区，在 Visual 模式下选择区域后输入 `:`，Vim 即可自动补全为 `:'<,'>` 。`:'<,'>s/foo/bar/g`
- 某行开始到某行`:5,12s/foo/bar/g`

[在 Vim 中优雅地查找和替换](https://harttle.land/2016/08/08/vim-search-in-file.html)

- `guu/gugu` 整行变为小写字母
- `gUU/gUgU` 整行变为大写字母

# 重复输入

比如要重复一个词100次，可以利用`100i` 的重复特性，会在退出`insert mode`的时候重复输入100次

1. 输入 `100i` 进入insert mode
2. 做一次录制输入
3. Esc退出重复100次操作，完成

### normal模式启用tab和shiftTab，通过修改keybinding.json实现

默认tab不在editorReadonly启用，即visual模式不能tab，拷贝如下配置到keybinding.json实现

```json
{
  "key": "tab",
  "command": "tab",
  "when": "editorFocus && !editorTabMovesFocus"
},
{
  "key": "shift+tab",
  "command": "outdent",
  "when": "editorFocus && !editorTabMovesFocus"
},
```

### kj退出insert模式

修改settings.json实现

```json
// ----------------------------------------------------------------------
// VsCode VIM
// https://marketplace.visualstudio.com/items?itemName=vscodevim.vim
// insert mode 连续 kj时退出 insert mode 
"vim.insertModeKeyBindings": [
  {
    "before": ["k", "j"],
    "after": ["<Esc>"]
  }
],
"vim.easymotion": true,
"vim.useCtrlKeys": true, // 允许insert模式里 control + <key> 快捷键的使用
"vim.useSystemClipboard": true, // 使用系统剪贴板而不是寄存器
// "vim.hlsearch": true,
// "vim.incsearch": true,
```
